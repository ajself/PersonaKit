# Idea Scorecard

Status: Active  
Owner: AJ  
Last Reviewed: 2026-03-07

## Why this exists

This scorecard keeps prioritization consistent when evaluating multiple
opportunities.

## Scoring Scale

- `1`: weak / unclear
- `3`: moderate / plausible
- `5`: strong / evidence-backed

## Weighted Criteria

| Criterion | Weight | What to look for |
| --- | --- | --- |
| User pain clarity | 3 | Clear problem users already feel |
| Time to MVP | 2 | Can we ship a useful MVP in 14 days? |
| Revenue path | 3 | Credible pricing and buyer path |
| Differentiation | 2 | Better than current alternatives |
| Distribution fit | 2 | Realistic channel to reach users |
| Technical feasibility | 2 | Fits current team capability |
| Ethics and trust fit | 3 | Aligns with [Ethics.md](./Ethics.md) |

## Formula

`Weighted Score = sum(score x weight)`

Maximum score: `85`

## Decision Bands

- `60-85`: Proceed to MVP outline.
- `45-59`: Keep in research queue and gather evidence.
- `<45`: Park for later; document why.

## Evidence Requirement

Each scored item must include at least one evidence note:

- user quote
- competitor signal
- market data point
- technical constraint discovered in implementation
