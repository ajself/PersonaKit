# Orbit Attempt Improvement Loop

Use this essential when Orbit is being rerun as a learning exercise instead of
as a one-off implementation sprint.

## Principle

Each Orbit attempt should inherit the last attempt's lessons explicitly and
leave better starting conditions for the next attempt.

Orbit should improve by accumulation, not by forgetting.

## Required Inputs From The Prior Attempt

Before a new attempt starts, load and summarize:

1. the latest rerun-prep note
2. the latest canonical retrospective
3. the latest retrospective comparison decision when method choice is relevant
4. the latest evidence packet
5. the unresolved code, product, process, and persona-fidelity issues

## Required Outputs From The Current Attempt

Each attempt should leave behind:

1. an evidence packet for the attempt
2. a product acceptance checklist result
3. an interaction-quality review artifact
4. participant-evidence artifacts for every active role used to justify the run
5. scoped confidence for:
   - feature outcome
   - product outcome
   - process outcome
   - persona-fidelity outcome
6. hybrid retrospective artifacts
7. an updated rerun-prep note or successor prep note that carries forward:
   - lessons to keep
   - lessons to reject
   - unresolved questions
   - next attempt start conditions

## Improvement Rules

1. Promote only evidence-backed lessons into the next attempt.
2. Keep disagreements visible instead of smoothing them away.
3. Preserve failed hypotheses when they explain why a later correction exists.
4. Distinguish between:
   - product improvement
   - process improvement
   - persona-orchestration improvement
5. If a later attempt changes the active prep note materially, create a new
   dated successor note instead of erasing the older decision trail.
6. Create new attempt-specific evidence files instead of rewriting an earlier
   attempt's packet set.

## Non-Goals

- Do not rewrite history so the previous attempt looks cleaner than it was.
- Do not collapse strong solo execution into a claimed multiagent success.
- Do not treat retrospective action items as complete until the next attempt
  actually absorbs them into its startup contract.
