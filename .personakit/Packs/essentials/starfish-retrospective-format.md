# Starfish Retrospective Format

Use the Starfish format as the default retrospective structure for
Samwise-led squad loops unless AJ explicitly requests a different format.

## Why This Format Is Default

Starfish is preferred when the work includes both:

- product or design delivery
- process, persona, or collaboration-model evaluation

This repo especially needs a format that can separate:

- what should continue unchanged
- what should increase
- what should decrease
- what should stop
- what must begin next

That makes Starfish a better default than simpler "went well / did not go well"
formats when the assignment is trying to judge both:

- the shipped feature
- the execution model that produced it

## Required Buckets

Every Starfish retrospective should include:

1. `Keep Doing`
2. `Less Of`
3. `More Of`
4. `Stop Doing`
5. `Start Doing`

## Samwise-Specific Guidance

When Samwise runs or synthesizes a retrospective, use Starfish to make the
following distinctions explicit:

1. feature delivery success versus process-experiment success
2. active persona execution versus advisory persona role-play
3. validated strengths versus flattering narration
4. bounded next-iteration actions versus broad process drift

## Orbit Guidance

For Orbit-specific work, treat Starfish as the preferred format because Orbit
retrospectives often need to cover both:

- command-center product quality
- persona, memory, and boundary fidelity

This format is especially useful when the result is mixed and needs rigorous
sorting instead of a simple positive/negative summary.
