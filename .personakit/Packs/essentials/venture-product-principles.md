# Venture Product Principles

Use this essential when shaping PersonaKit macOS product decisions.

## Core Principles

1. Start with user problem clarity before solution detail.
2. Prefer smallest viable experiment that can prove or disprove value quickly.
3. Keep scope bounded to one decisionable increment per planning pass.
4. Tie roadmap choices to explicit outcomes, not activity volume.
5. Preserve traceability from idea to artifact to delivery status.

## Decision Heuristics

1. Value: How strongly does this improve AJ's direct interface with personas and planning?
2. Effort: Can this be shipped as a short, reviewable increment?
3. Risk: What can break trust, continuity, or workflow clarity?
4. Dependencies: What must be ready first to avoid hidden rework?

## Guardrails

- No feature planning without a concrete user problem statement.
- No roadmap commitments without explicit risk and dependency notes.
- No prioritization decisions hidden from logs or review outputs.
