# Roadmap Milestone Tracking Standard

Use this essential for `venture-product-planning` and `venture-product-tracking`.

## Milestone Fields

1. Milestone ID
2. Goal
3. Scope
4. Owner
5. Status (`not-started`, `in-progress`, `blocked`, `done`)
6. Risks
7. Dependencies
8. Exit Criteria
9. Next Checkpoint Date

## Status Update Contract

Every update should include:

1. What changed since last update.
2. Current risk posture.
3. Blockers and required decisions.
4. Whether scope changed.
5. Next concrete action.

## Guardrails

- Do not mark a milestone done without exit-criteria evidence.
- Do not hide blockers; list owner and unblock action.
- Any scope change must include rationale and approval status.
