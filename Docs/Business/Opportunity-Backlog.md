## Purpose

Track opportunity candidates in one place with scoring, state, and next action.

## Status Definitions

- `Exploring`: idea exists, not scored yet
- `Scored`: evaluated with scorecard, waiting decision
- `Selected`: chosen for current 14-day MVP cycle
- `Parked`: intentionally paused
- `Dropped`: closed with rationale

## Daily Update Checklist

1. Add new opportunities from the current session run.
2. Score candidates using [Idea-Scorecard.md](./Idea-Scorecard.md).
3. Update status and next action.
4. Capture one reason for each park/drop decision.
