# Milestone Templates

Status: Draft
Owner: Samwise
Last Updated: 2026-03-18

## Purpose

Provide a reusable template library for later Orbit milestone dossiers.

These templates are derived from the stronger patterns established in `M0` and
`M1`. Their job is to make later milestone planning more disciplined,
reviewable, and quality-aware.

## How To Use These Templates

- copy the relevant template into the milestone directory before editing it
- replace placeholders with milestone-specific content immediately
- delete sections that are truly irrelevant only if the omission is explicit and
  justified
- do not leave generic boilerplate in a live dossier

## Quality Rule

These templates exist to raise the quality bar, not to automate paperwork.

If a milestone file still reads like a generic shell after filling it out, the
template has not been used correctly.

- `Milestone-Decision-Register-Template.md`
- `Milestone-Persona-Coverage-Matrix-Template.md`

## Recommended Use Pattern

- use the core templates for every milestone that is ready for serious execution
- add the specialized templates only when the milestone genuinely needs them
- always keep quality, evidence, stop points, and review gates explicit

## Anti-Pattern

Do not copy every template into every milestone folder by default.

Use the smallest set that still makes the milestone impressive, reviewable, and
hard to misunderstand.
