# Retrospectives

Status: Active  
Owner: AJ + Samwise  
Last Reviewed: 2026-03-08

## Purpose

Store durable retrospective records and recommendation follow-ups for delivery
loops that should continue informing future work after a plan closes.

Retrospectives are not just wrap-up notes. They are evidence-bearing closeout
artifacts that should make feature outcome, product outcome, and process
outcome visible without blending them together.

## Working Defaults

1. Use `Starfish` by default unless AJ asks for a different retrospective
   shape.
2. Use `hybrid` by default when persona fidelity, process quality, or
   product-bearing milestone closeout is under review:
   - `fan-out` first
   - `roundtable` second
   - one canonical Starfish synthesis at the end
3. Record declared roles, actual participants, participant evidence, and
   feature/product/process confidence explicitly.
